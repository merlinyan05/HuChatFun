# HuChatFun 数据清洗策略说明

## 核心问题：原始语料的噪声结构

从你提供的两条直播文字稿分析，内容大致分布如下：

| 内容类型 | 占比 | 处理方式 |
|----------|------|----------|
| 感谢礼物/读SC/念名单 | ~40% | **全删** |
| 等人/技术问题/催促连麦 | ~20% | **全删** |
| 连麦对话（无实质内容的） | ~15% | **删** |
| 连麦对话（有观点输出的） | ~15% | **保留+改造** |
| 独白式观点输出（金句） | ~10% | **重点保留+增强** |

也就是说，**一条几万字的文字稿，真正有训练价值的内容大概只有 10-15%**。

---

## 你想要的效果：口头禅更密集、观点更爆

这不能光靠清洗，需要三管齐下：

### 1. 数据层：只喂"精华"给模型

不要把完整的直播喂给模型。只喂经过筛选的高密度观点段落。模型学到的就是"每句话都在输出观点"的户晨风，而不是"60%时间在说谢谢、等一下、不着急"的户晨风。

### 2. System Prompt 层：显式要求高密度

在 system prompt 里直接列出口头禅清单，并要求模型高频使用。这比让模型自己从数据里学要可控得多。

### 3. 风格增强：用 LLM 改写训练数据

拿到户晨风的原始观点后，用另一个 LLM（比如 Claude）改写成口头禅密度更高的版本。相当于做一个"浓缩液"。

---

## 具体清洗步骤

### 步骤一：物理去噪

把以下模式直接正则删除（详见 pipeline 代码）：

```
感谢xxxx总 / 感谢xxxx总的舰长 / 再次感谢...  → 删
好，不着急啊，不着急 / 稍安勿躁...          → 删
读一下SC / 再对一下SC...                    → 删
你太卡了 / 安卓网 / 先给对方闭麦...          → 删
点左上角 / 有个加号 / 开直播...（操作指导）    → 删
xxxx总xxxx总xxxx总...（结尾念名单）          → 删
```

这一步能砍掉 50-60% 的文字量。

### 步骤二：按连麦切分对话

用以下标志切分独立对话：
- **开始标志**："你好，请讲" / "直接表达观点" / "喂"
- **结束标志**："好，再见" / "拜拜" / "下一个"

### 步骤三：对话质量评分

给每段对话打 1-5 分，只保留 ≥3 分的：

**加分项：**
- 包含高价值信号词（"我告诉你""纯纯""安卓""购买力"等）
- 户晨风有实质性观点输出（200-1000字）
- 有明确的话题和结论

**扣分项：**
- 对方没说实质内容就挂掉的
- 大量技术问题（卡、闭麦、开直播教学）
- 纯闲聊无观点

### 步骤四：从你的两条语料中，高价值段落示例

**必须保留的金句段落：**

1. **苹果拍照论**："所有手机的拍照都在跟iPhone比，只有iPhone在跟肉眼比"
2. **不锈钢筷子论**：整段关于木筷子发霉→黄曲霉素→换304不锈钢
3. **电焊+雅思论**："电焊加雅思6.5大于等于985"
4. **ASML选择论**：先去头部企业→曲线跳槽→收入更高
5. **菜市场论**：见人下菜+缺斤少两→去连锁超市
6. **农村自建房论**：不会保值升值→去大城市
7. **洗碗机论**：离不开水离不开空气离不开洗碗机
8. **换电模式论**：换电池=换心脏→不知道好坏
9. **中医/健胃消食片对话**：整段连麦辩论很精彩
10. **500万离婚辩论**：与日本留学生的长对话

**应该删除的低价值段落：**

1. 所有"检查房间/扔东西"的细节操作指令（保留结论即可）
2. 等对方开直播/调网络的过程
3. 上麦后一句话没说就挂掉的
4. 反复念感谢名单

### 步骤五：构造训练格式

**对话型数据（连麦精华）：**

```json
{
  "conversations": [
    {"role": "system", "content": "[system prompt]"},
    {"role": "user", "content": "户子，iPhone 17 值得买吗？"},
    {"role": "assistant", "content": "值得购买。但对于你来讲现在这个负担有点大了..."}
  ]
}
```

**观点型数据（独白金句 → LLM 生成配对问题）：**

用 Claude 或 Qwen 把户晨风的独白观点自动配上一个合理的"用户提问"。

### 步骤六：风格增强

用 LLM 把原始回答改写成口头禅密度更高的版本：

```
原文: 你这个工作收入不高，建议你考虑换行业。
增强: 你一个月挣这点钱，纯纯安卓收入，购买力约等于零。
     我告诉你这行业你待下去纯废，知道吧？赶紧换，
     别搁这耗着了，耗到最后前程似锦。能明白吗？
```

## 数据量预估

| 来源 | 预估数量 | 说明 |
|------|----------|------|
| 连麦精华对话 | 每场 5-10 条 | 一场直播约 20 次连麦，精华率 30-50% |
| 独白金句 | 每场 10-20 条 | 读SC回复、话题展开 |
| LLM 增强版 | 上述 x2 | 每条原始数据生成 1 条增强版 |

按 2023-2025 约 500 场直播估算：
- 原始精华：约 5000-10000 条
- 增强后：约 10000-20000 条

这个量级足够让 7B 模型学到稳定的风格。

---

## 验证清洗质量的方法

清洗完后，随机抽 50 条训练数据，检查：

1. **口头禅密度**：每条回答中至少出现 2-3 个标志性口头禅
2. **观点明确度**：每条回答都有明确的判断/建议，不是废话
3. **无噪声**：没有"感谢xx总""不着急""读一下SC"等废话
4. **长度合理**：回答在 50-500 字之间（太短没风格，太长有噪声）
5. **问答对齐**：用户的问题和户晨风的回答是对应的
